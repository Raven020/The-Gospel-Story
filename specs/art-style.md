# Art Style & Sprite Design Specification

## Target Style
GBA-era Pokemon (Fire Red / Emerald generation). Top-down overworld perspective with clean pixel art.

---

## Pixel Art Principles

### Palette Constraints
- 15 colors + 1 transparent per sprite palette
- Every color earns its place — no wasteful gradients
- Each character uses a cohesive palette of ~8-12 colors
- Shadows and highlights are **hue-shifted**, not just darker/lighter
  - Skin shadow leans warm (not just dark peach)
  - Robe shadow shifts slightly cool
  - Hair shadow deepens in hue, not just value

### Sprite Dimensions
- **Overworld characters:** 16x32 pixels (width x height)
- **Map tiles:** 16x16 pixels
- **Small objects/animals:** 16x16 pixels (e.g., Dove)
- **Battle sprites:** 64x64 pixels (front-facing, Pokemon Fire Red style)

### Readability
- Strong silhouette — character identifiable from shape alone at 1x
- High contrast between character and background
- Key features exaggerated (hair, clothing, accessories) since pixels are scarce
- 1px dark outline around the entire sprite for separation from backgrounds

### Character Proportions
- Head is ~40% of total sprite height (large for expressiveness, Pokemon style)
- Eyes are simple — 1-2 pixels, but convey personality through placement
- Clothing uses flat color blocks with 2-3 levels of shading
- Details are suggested, not drawn — a sash is a single row of darker pixels

### Shading
- 2-3 levels per color: base, shadow, highlight
- Consistent light source: **top-left**
- Hard edges between shade levels (cel-shaded look), no dithering on characters
- Dithering reserved for backgrounds/environments if needed

## Directional Sprites

Each character requires 3 static sprite variants for 4-directional movement:

| Direction | Sprite | Notes |
|-----------|--------|-------|
| **Front (Down)** | Unique art | Facing camera — full face, most detail |
| **Back (Up)** | Unique art | Walking away — all hair/clothing from behind, no face |
| **Left** | Unique art | Profile view — mirrored in code for Right |
| **Right** | Left mirrored | Flipped horizontally via `renderSpriteMirrored()` — no extra art needed |

### Front View Rules
- Full face visible: eyes, nose shadow, mouth/beard
- Hair frames face on both sides
- Body is widest in this view (arms/sleeves at sides)
- Most visual detail and character personality here

### Back View Rules
- No face visible — entire head is hair from behind
- Hair texture uses **darker tones** than front (HD/HM, fewer HL highlights)
- Robe uses more **shadow tones** (RS) instead of highlights (RH) — back is away from light
- Same body width and silhouette as front for consistency
- Sash visible from behind at same row position

### Side (Left) View Rules
- **Profile face**: one eye visible, nose protrudes as a distinct skin pixel bump
- **Clean face area**: no hair pixels overlapping the face — skin runs cleanly on the face side, hair only on the back-of-head side
- Mouth/beard visible from the side
- Hair falls behind on the far side (right columns)
- Body is **narrower** than front/back — turned sideways
- Arms: one arm visible as sleeve on each side of the narrower body
- **Legs/feet stay the same 2px-wide sandal style** as front view — not slimmed down
- One foot slightly forward for walking stance

### Mirroring (Right from Left)
- Right-facing sprite is generated by flipping Left horizontally in code
- No separate art asset needed
- `renderSpriteMirrored()` reverses column order during rendering

- 3-4 frames per direction
- 4 directional facings: down, up, left, right
- Left sprite can be mirrored for right to save asset work

- Slight vertical bob on walk for lively feel (Pokemon bounce)

### Special Animations
- Baptism transformation glow
- Transfiguration radiance
- Scripture-selection combat poses

## Character Design Philosophy

### Silhouette-First Design
Every character must be distinguishable by shape alone:
- **Peter:** Stocky, broad shoulders, big bushy beard
- **John:** Slim, youthful, clean-faced
- **Judas:** Narrow, pinched, rat-like — visually distinct from all others
- **John the Baptist:** Wild, wide (big hair), rugged
- **Satan:** Tall, angular, imposing
- **Angel:** Wings break the silhouette, ethereal tapered feet
- **Young Jesus:** Short, large head-to-body ratio (child proportions)

### One Defining Feature Per Character
With limited pixels, each character gets one standout visual trait:
- Mary: Blue veil
- Joseph: Grey-streaked hair, broad build
- John the Baptist: Wild untamed hair, camel garment
- Satan: Crimson accents, pale skin, red-tinted eyes
- Angel: Wings, golden hair, glow pixels
- Matthew: Maroon/rich robe (wealth)
- Thomas: Medium, unassuming (his "doubt" is internal, not visual)
- Thaddaeus: Auburn/red hair
- Simon the Zealot: Intense dark presence, thick beard

### Brother Pairs Look Similar
Siblings share hair color family and robe style to communicate relation:
- **Peter & Andrew:** Reddish-brown hair, fisherman blue-grey robes. Peter is stockier with a bigger beard; Andrew is slimmer with a shorter beard.
- **James & John (Zebedee):** Dark brown/black hair, fisherman blue-grey robes. James is sturdier with a full beard; John is youngest, no beard.

### Judas — Intentionally "Off"
Judas is designed to look visually wrong compared to the other disciples:
- **Narrower head** (10px wide vs 12px for others)
- **Close-set eyes** placed toward center of face
- **Scraggly thin beard** (sparse, not full)
- **Greasy dark hair** (muted, yellowish-dark, not clean brown or black)
- **Dirty green-brown robe** (no other character uses this color)
- **Narrow hunched shoulders** (less pixels wide than any other adult male)
- **Skinny legs and smaller feet**
- Observant players should feel something is "off" about him before the story reveals it

## Skin Tones

All characters use the same Middle Eastern skin tone palette for consistency:

| Role | Base | Shadow | Highlight |
|------|------|--------|-----------|
| Standard | `#D4A574` | `#B8845A` | `#E8C49A` |
| Weathered (John Baptist) | `#C89A60` | `#A87A48` | `#DEAE72` |
| Pale/unnatural (Satan) | `#C8B8A8` | `#A89888` | `#DED0C0` |
| Luminous (Angel) | `#F0DCC8` | `#D8C4B0` | `#FFF0E0` |

Skin tone variation is minimal and reserved for supernatural or story-specific reasons, not for differentiating human characters.

## Sprite File Organization

All sprite data lives in `specs/sprites/` as ES modules:

| File | Contents |
|------|----------|
| `jesus-overworld.js` | Adult Jesus (front-facing) |
| `young-jesus.js` | 12-year-old Jesus |
| `mary-joseph.js` | Mary and Joseph |
| `john-baptist.js` | John the Baptist |
| `satan.js` | Satan |
| `angel-dove.js` | Angel + Dove (Holy Spirit) |
| `disciples.js` | All 12 disciples |
| `pharisees.js` | 5 Pharisee NPC variants |
| `townspeople.js` | Generic townspeople + Temple Teacher |
| `preview.html` | Visual preview page (serves all sprites at 8x scale) |

Each file exports:
- A `PALETTE` object mapping color keys to hex values
- One or more sprite data arrays (16-wide arrays of palette key strings, 32 rows)
- A `renderSprite()` function for canvas rendering

## Rendering

### Internal Resolution
- Fixed GBA-style internal resolution (240x160 or similar)
- Scaled responsively to fill the browser window
- **Nearest-neighbor scaling** — no anti-aliasing, no blur
- `image-rendering: pixelated` / `crisp-edges` on all canvases
- Maintain aspect ratio at all times

### Sprite Rendering
- Sprites rendered pixel-by-pixel to canvas via `fillRect` calls
- Transparent pixels (`_` / `null`) are skipped, not drawn
- Scale factor applied uniformly to both dimensions
- No sub-pixel rendering — all positions snapped to pixel grid
